# Engineering Standards

## Standard 1: Bounded Scope

Every artifact must state what it does and does not do.

## Standard 2: Reproducibility

A reviewer must be able to reproduce the setup, execution, and validation steps from repository content.

## Standard 3: Secure Defaults

Workflows and examples should use least privilege, explicit permissions, and no committed secrets.

## Standard 4: Reviewable Evidence

Validation should produce reviewable evidence such as logs, workflow artifacts, reports, or markdown summaries.

## Standard 5: Human Authority

AI-generated output is advisory. Human review is required before merge, deployment, release, or compliance assertion.

## Standard 6: Clear Promotion Criteria

A lab can become an accelerator only after repeatable validation, risk review, and evidence capture.

## Standard 7: Minimal Operational Assumptions

Labs should not depend on hidden local state, undocumented accounts, implicit credentials, or production-only configuration.

## Standard 8: Explicit Failure Modes

Labs should document known limitations, misuse cases, and failure modes.
